## Vector DB Use Case

A traditional keyword-based search is not enough for large legal contracts. It only searches for exact words, so it may miss important information if different terms are used. For example, a termination clause might be written as “contract end” or “agreement cancellation,” which keyword search may not find.

A vector database works differently. It understands the meaning of text using embeddings. It converts both the contract and the user’s question into vectors and compares them based on similarity. This helps find relevant information even if the exact words are not the same.

In this system, the contract is divided into smaller parts, and each part is converted into an embedding. When a lawyer asks a question, the system finds the most similar parts of the contract and shows them as results.

This makes searching faster and more accurate. Therefore, a vector database is very useful for understanding and searching large legal documents, which is not possible with simple keyword search.
